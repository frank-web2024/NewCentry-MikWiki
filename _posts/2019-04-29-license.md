服务器性质

本服务器为创造休闲向服务器，以建筑创作与社区交流为核心。服务器对客户端 Mod 使用持开放态度，但一切游戏行为均须以尊重他人创作、维护服务器秩序为前提。

## 玩家身份与考察制度

新玩家加入后默认为**黄名**身份，处于考察期。考察期内管理员会对玩家的行为进行观察。

考察期无固定时长，系统将根据玩家的活跃度进行计算并自动晋升为正式成员（**白名 Member**），具体算法不对外公开。若在考察期内出现违规或不当行为，管理员有权延长考察期限。

- **黄名**：新玩家默认身份，处于考察阶段，具有以下权限限制：
  - Axiom 功能受限
  - 无法在聊天中使用格式化文本
  - 无法使用装饰头颅获取命令 `/headdb`（或 `/hdb`）
  - 无法使用服务器状态查询命令 `/mspt`、`/tps`
  - 无法使用清除周围掉落物命令 `/rmitems`
  - 无法使用临时白名单命令 `/tempwhitelist`
  - 无法使用前后缀设置命令 `/nametag`
  - 无法使用原版命令 `/attribute`、`/damage`、`/effect`、`/fill`、`/item`、`/ride`、`/save-all`、`/setblock`、`/summon`
- **白名 Member**：考察通过后的正式成员身份，由系统根据活跃度自动晋升，解除上述限制

## 建筑与创作规范

1. **禁止破坏他人建筑**，禁止破坏建筑必要之外的地形。反例：在野外随意挖坑；将山体削去一半后不做平滑处理。
2. **建筑需与周围环境协调**，避免明显断层或突兀结构，选址需合理。反例：在植被茂密的森林中央平地建造大型混凝土方块建筑而不做任何过渡；在已有建筑密集的区域紧贴他人建筑建造风格完全冲突的结构。最终是否合适由管理员判断。
3. **允许轻量红石**，禁止大规模或卡服红石装置，尽量降低高频运算。漏斗（Hopper）会持续产生性能开销，建议避免使用，尤其禁止大量堆叠。如有疑问请提前询问管理员。
4. **必须在建筑中合理位置标注建筑者 ID 及原作者**（如有参考或复现他人作品）

## Mod 与外挂使用规范

服务器允许玩家自由使用客户端 Mod，包括通常意义上的辅助类"外挂"功能，但以下行为无论通过何种方式实现均**明确禁止**：

1. **对其他玩家发动攻击、强制位移或造成非自愿传送**，干扰其正常游戏
2. **破坏、移动、覆盖或修改他人的建筑或其所依托的地形**，包括被纳入建筑设计的自然地形
3. **绕过服务器权限限制**，包括执行越权指令、强制改变他人游戏模式等
4. **导致服务器性能下降的一切行为**，包括短时间内大量生成实体或掉落物、触发大范围持续区块加载、高频重复执行大规模操作等

> 简而言之：Mod 的使用以不影响他人、不破坏服务器为界限。如对某一用法是否合规存有疑问，请提前询问管理员。

## 行为与社区规范

1. **保持基本礼貌**，禁止在公共频道发表侮辱、攻击、骚扰其他玩家的言论
2. **禁止人身攻击与恶意挑衅**，包括针对玩家的建筑风格、审美或个人身份的恶意评价
3. **禁止在任何公共场合大量发送重复或无意义内容**，包括聊天频道、告示牌、书与笔等，干扰他人正常交流
4. **禁止发布与服务器无关的广告、引流或商业推广内容**
5. **禁止散布虚假信息或恶意谣言**，尤其是涉及服务器管理、其他玩家的不实内容

3. 允许在非商业用途下复制或改编作品，但必须注明原作者及 Mik 米客服务器名称
4. 禁止将作品用于盈利服务器、付费内容、地图售卖或其他商业用途
5. 衍生或改编作品必须继续采用 CC BY-NC-SA 4.0 许可发布
6. 禁止未经许可打包发布服务器存档或大规模搬运服务器整体建筑内容；此条为对 CC 协议的附加限制，优先适用
7. 服务器保留对作品进行展示、宣传及社区推广的使用权

## 官网建筑收录规范

服务器官网将不定期收录服务器内的优质建筑作品，以展示社区创作成果。收录遵循以下规则：

1. **作品须具备一定完成度，且符合服务器建筑与创作规范**，未完成或存在违规的建筑不予收录
2. **作品须为创作者本人所有**，多人合作作品须列明所有主要参与者
3. **收录前须取得原作者明确同意**，作者有权拒绝收录或申请撤除已收录的作品；作者申请撤除时，管理员应在合理时间内予以处理。若作品涉及多位创作者且意见不一，由管理员协调处理
4. **展示内容须注明建筑者 ID 及创作时间**，多人合作作品须注明所有主要参与者
5. **禁止展示含有侵权内容的建筑**，包括未经授权复现的受版权保护的 IP 形象或建筑等
6. **收录内容遵循 CC BY-NC-SA 4.0 许可协议**，官网对作品的展示与推广属于协议允许范围内的非商业性使用

> 管理员拥有对上述规则的最终解释权，并有权根据实际情况作出判断与处置
